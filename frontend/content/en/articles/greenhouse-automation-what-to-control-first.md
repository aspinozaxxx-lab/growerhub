It is better to start automating a greenhouse not with the maximum set of devices, but with a risk map. What can ruin plants the fastest: overheating, drying out, overwatering, leaking, forgotten watering, ventilation failure or lack of responsibility? When priorities are clear, GrowerHub becomes a control system rather than a sensor showcase.

For a small greenhouse or mini-farm, it is important to divide the task into zones. Each zone has its own plants, sensors, rules and log. If everything is mixed into one page, automation quickly becomes unclear: the alarm has arrived, but it is unclear where the problem is and who should respond.

## First level: security

The most important thing is accidents. Leaks, overheating, frozen pump, lack of data from the sensor and loss of power should be visible immediately. Complex forecasts are not needed here. We need clear stop conditions and notices. If water leaves the system, the pump turns off. If the temperature is above the permissible level for too long, the person in charge receives a signal.

It is better to build security locally. The cloud is great for remote viewing, but basic stops need to work close to the hardware. This is especially important for watering and ventilation.

## Second level: monitoring

After security, set up monitoring: temperature, air humidity, substrate humidity, light, condition of pumps, sockets and sensors. You don't have to measure everything at once. It is important that each parameter is associated with the solution. If there is a sensor, but no one understands what to do in case of an alarm, its usefulness is limited.

For several boxes or beds, use the same card structure: zone, current values, latest events, accidents, responsible. This is described in detail in the article [monitoring multiple boxes](/articles/monitoring-neskolkih-boksov).

## Third level: management

Automatic control should be turned on after observation. First let GrowerHub show graphs and notifications, then add actions: watering, ventilation, light, service locks. Each action must have limits and a log. If the pump turned on, there should be a record: why, for how long, which sensor was involved, whether the limit was triggered.

For microclimate, do not strive for accuracy to the tenth. It is more important that the system shows repeating scenarios: overheating during the day, high humidity at night, a rapid decrease in humidity after ventilation. The basics are in the article [microclimate control in a greenhouse](/articles/kontrol-mikroklimata-v-teplitse).

## Fourth level: accounting

When the basic system is running, a management layer appears: water flow, number of irrigations, maintenance, yield, user roles. This is needed not only by business. Even a home mini-farm becomes easier if you understand how much water has been used, which areas require attention and which mistakes are repeated.

It is better to start accounting for water and irrigation early, because without history it is impossible to compare seasons. The article [accounting for water and irrigation in a greenhouse](/articles/uchet-vody-i-polivov-v-teplitse) shows what data is needed.

## Conclusion

The correct order of greenhouse automation: safety, monitoring, management, accounting. Don't start with a large set of scenarios. Start with zones, alarms, latest data and log. Then GrowerHub helps manage the process rather than just showing a lot of numbers.
