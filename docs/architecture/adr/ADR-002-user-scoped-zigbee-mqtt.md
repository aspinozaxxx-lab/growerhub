# ADR-002: пользовательская изоляция Zigbee MQTT

Статус: принято

## Контекст

Текущий Zigbee-контур использует один namespace `zigbee2growerhub`, один snapshot и административный API. Это не позволяет безопасно подключать координаторы разных пользователей и допускает пересечение IEEE-адресов, friendly name, зон и команд.

## Решение

- Один пользователь является владельцем нескольких координаторов, зон и автоматизаций без отдельной сущности организации.
- Координатор имеет публичный UUID и отдельную MQTT-учётную запись. Его namespace равен `gh/z2m/<mqtt_username>/#`.
- Backend разрешает операции только после проверки владельца координатора или зоны.
- Zigbee snapshot, история и команды идентифицируются парой `coordinator_id` и идентификатором устройства.
- Mosquitto использует TLS и Dynamic Security с запретом по умолчанию. Для каждого координатора создаётся отдельная роль с буквальным ACL `gh/z2m/<mqtt_username>/#`; MQTT client ID фиксируется равным `mqtt_username`. Общая роль с подстановкой `%u` не используется, потому что она не обеспечивает одинаково надёжную проверку публикаций и подписок в установленной версии Mosquitto.
- Пароль координатора формируется при создании или ротации, передаётся один раз и не хранится приложением в открытом виде.
- Legacy namespace `zigbee2growerhub/#` поддерживается через отдельную учётную запись MQTTS только на время переноса существующего координатора и не является публичным self-service контрактом.
- До обновления аутентификации существующего контроллера GROVIKA допускается отдельный брокер на `1883`, доступный только из доверенной локальной сети. Он принимает только `gh/dev/#` и двусторонне пересылает этот namespace в основной брокер через учётную запись backend. Публичный нешифрованный `1883` остаётся запрещённым.
- Для существующего локального Zigbee2MQTT разрешён connector с направленной пересылкой Z2M-состояний и команд. Локальные MQTT credentials не передаются backend.

## Последствия

- Домен `zigbee` начинает владеть координаторами и пользовательской изоляцией своего snapshot.
- Домен `automation` хранит владельца помещения и координатор в Zigbee-привязке.
- MQTT adapter извлекает identity координатора из topic и передаёт его в `ZigbeeFacade`.
- Внешнее управление учётными записями broker оформляется шлюзом домена `zigbee`, реализация остаётся в MQTT adapter.
- Публичный запуск self-service разрешён после миграции legacy-данных, проверки ACL, перевода Zigbee2MQTT на MQTTS и изоляции временного доступа GROVIKA локальной сетью.
