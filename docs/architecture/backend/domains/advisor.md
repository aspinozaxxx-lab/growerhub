# Домен advisor

## Назначение

Формирует рекомендации по поливу растения на основе данных растения, последнего полива, истории метрик и настроек advisor.

## Публичный Facade

`AdvisorFacade`

- `getWateringAdvice(Integer plantId, AuthenticatedUser user)`

## Публичные контракты

- `WateringAdvice`
- `WateringAdviceBundle`
- `WateringAdviceGateway`
- `WateringPrevious`

## Владение данными

Домен владеет кэшем рекомендаций `advisor_watering_advice`. Данные растения, истории и журнала не хранит как свои, а получает через публичные Facade и contracts других доменов.

## Используемые домены

- `plant`
- `journal`

## Внешние пользователи домена

- REST adapter `api`
- пакет внешнего шлюза `llm`

## Алгоритм работы

Facade проверяет включение advisor, получает растение, последний полив и bucket-историю метрик. Затем собирает контекст и передает его в engine. Engine возвращает рекомендацию из кэша или вызывает внешний шлюз и сохраняет результат.

## Ограничения

Advisor не владеет растениями, журналом и метриками. Внешний LLM вызывается только через контракт `WateringAdviceGateway`. Параметры advisor должны приходить из конфигурации.
