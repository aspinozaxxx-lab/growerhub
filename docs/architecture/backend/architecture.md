# Backend

Backend построен вокруг доменов. Адаптеры REST и MQTT принимают внешние сообщения, выполняют технический маппинг и вызывают публичные операции доменных Facade.

## Пакеты

- `api` - REST controllers, REST DTO и маппинг ошибок.
- `mqtt` - MQTT-издатель, подписчик, обработчики, шлюзы и модели сообщений.
- `common` - стабильные общие контракты, утилиты и конфигурационный wiring.
- `config` - общая конфигурация приложения.
- `diagnostics` - техническая диагностика.
- `llm` - внешний шлюз для advisor.
- доменные пакеты - `advisor`, `auth`, `device`, `firmware`, `journal`, `plant`, `pump`, `sensor`, `user`, `zigbee`.

## Домены

Фактические домены backend:

- `advisor` - рекомендации по поливу.
- `auth` - аутентификация, токены, методы входа и SSO.
- `device` - устройства, shadow, ACK и service events.
- `firmware` - хранение и запуск обновлений прошивки.
- `journal` - журнал растений и детали полива.
- `plant` - растения, группы, жизненный цикл и метрики.
- `pump` - насосы, привязки и ручной полив.
- `sensor` - датчики, привязки, текущие показания и история.
- `user` - пользователи, профиль и администрирование.
- `zigbee` - админский snapshot Zigbee2MQTT bridge и командный шлюз для Zigbee-устройств.

Описание каждого домена находится в `docs/architecture/backend/domains/`.

## Границы

У каждого домена есть один публичный Facade в корне доменного пакета. Внешний код обращается к домену через Facade и типы из `contract`.

Внутренние пакеты `engine` и `jpa` принадлежат только своему домену. Другие домены и адаптеры не импортируют чужие `engine` и `jpa`.

Домен не импортирует `api` и `mqtt`. Адаптеры не импортируют JPA entity и repository.

`common` не содержит JPA entity и не владеет бизнес-данными.
