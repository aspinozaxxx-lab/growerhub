# Frontend

Frontend - React-приложение. Оно отображает состояние из backend и не является источником правды.

## Структура

- `src/main.jsx` - точка входа.
- `src/App.jsx` - маршрутизация.
- `src/api` - HTTP client и API-модули.
- `src/components` - переиспользуемые UI-компоненты.
- `src/content` - статический контент страниц и статей.
- `content/equipment` - единый каталог рекомендованного оборудования с датой проверки и внешними ссылками.
- `src/domain` - локальные frontend-типы и утилиты, не равные backend-доменам.
- `src/features` - прикладные состояния, hooks и contexts.
- `src/pages` - публичные страницы.
- `src/pages/app` - страницы приложения.
- `src/utils` - общие утилиты.

## Состояние

`AuthContext` хранит access token только в памяти и восстанавливает сессию через защищённую refresh-cookie. SSO callback не передаёт токен в URL. `SensorStatsContext` и `WateringSidebarContext` управляют боковыми панелями. Данные dashboard, sensors и watering загружаются через hooks и API-модули.

## Публичный сайт и self-service

Публичные страницы, включая `/kak-nachat/` и каталог `/oborudovanie/`, статически генерируются из тех же content-моделей, которые использует React. Для индексируемого маршрута H1, основной текст и CTA должны совпадать до и после выполнения JavaScript. Canonical, Open Graph, JSON-LD и внутренние ссылки используют конечный URL с `/`.

Приложение, административные страницы и графики загружаются лениво и не входят в начальный публичный bundle. Маршруты кабинета self-service находятся под `/app/`: onboarding, обзор, подключения, зоны, устройства, автоматизации и профиль. Растения остаются дополнительной функцией и не требуются для onboarding.

`VITE_SELF_SERVICE_ENABLED` управляет публичным входом в платформу. Пока флаг выключен, основной CTA ведёт на страницу раннего доступа без запуска регистрации. Сборка с включённым флагом запрещена, если в контенте приватности и условий использования не заполнены и не подтверждены данные оператора.

Продуктовые события Метрики отправляются через одну утилиту с фиксированным allowlist параметров. Email, внутренние идентификаторы, IEEE и MQTT credentials в аналитику не передаются.

## Границы

Frontend обращается к backend только через REST API. Прямой доступ к БД, MQTT broker и внутренним backend-пакетам запрещен.

Локальные MQTT credentials для connector вводятся пользователем только в браузере и могут попадать лишь в скачиваемый локальный конфиг. Frontend не отправляет их GrowerHub.
